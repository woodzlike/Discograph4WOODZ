## Phase 6. 다크모드 지원

> **왜 이 순서?** `next-themes`(^0.4.6)는 이미 설치되어 있고 `app/globals.css`에 `.dark` 클래스 기반 CSS 변수 토큰도 이미 정의되어 있다. 즉 토큰 자산은 준비된 상태이고 이를 활성화하는 Provider와 토글 UI만 없는, 가장 적은 리스크로 완료 가능한 작업이다. 시각적 인터랙션을 다루는 Phase 7(호버 3D 효과)보다 먼저 처리해 다크모드 상태에서도 새 인터랙션 효과가 잘 보이는지 함께 점검할 수 있도록 한다.

**예상 소요 시간**: 0.5일

### 작업 목록

- [ ] `app/layout.tsx`에 `next-themes`의 `ThemeProvider` 연결 — `attribute="class"`, `defaultTheme="system"`, `enableSystem` 설정
- [ ] 테마 토글 UI 컴포넌트 신규 작성 (`components/theme-toggle.tsx`) — 설치되어 있는 `components/ui/dropdown-menu` 또는 `components/ui/switch` 활용, 라이트/다크/시스템 전환
- [ ] `ArtistHeader`(`components/artist-header.tsx`) 영역에 테마 토글 버튼 배치 — 기존 레이아웃(프로필 이미지, 이름, 팔로워 수)을 깨지 않는 위치 선정
- [ ] Hydration mismatch 방지 처리 — `useTheme()` 사용 시 `mounted` 상태 가드 또는 `suppressHydrationWarning` 적용
- [ ] 다크모드 상태에서 기존 컴포넌트(`AlbumCard`, `StarRating`, `FilterTabs`, `Badge` 등) 시각적 점검 — 시맨틱 토큰(`bg-background`, `text-foreground` 등)만 사용했는지 재확인하고, 색상이 깨지는 부분이 있으면 보정
- [ ] 새로고침 후에도 선택한 테마가 유지되는지 확인 (localStorage 기반 영속성은 `next-themes` 기본 동작)

### 완료 기준

- 토글 클릭 시 라이트 ↔ 다크 모드가 즉시 전환된다
- 새로고침해도 마지막으로 선택한 테마가 유지된다
- 시스템 다크모드 설정을 따르는 "시스템" 옵션이 정상 동작한다
- 다크모드에서 메인 페이지의 모든 텍스트/배경/배지 대비가 깨지지 않는다 (하드코딩된 색상값 없이 시맨틱 토큰만 사용)
- 초기 페이지 로드 시 테마 깜빡임(flash) 또는 hydration 경고가 콘솔에 발생하지 않는다

## Phase 7. 앨범 카드 인터랙션 고도화 (호버 3D 입체 효과)

> **왜 이 순서?** 시각적 인터랙션 추가는 Phase 6에서 다크모드 토큰이 정리된 이후에 진행해야, 두 모드 모두에서 그림자·하이라이트 효과가 의도대로 보이는지 한 번에 검증할 수 있다. 또한 `AlbumCard`는 메인 페이지의 핵심 콘텐츠이므로 다른 보류 항목(상세 페이지 등)보다 사용자가 가장 자주 마주치는 영역에 우선 투자한다.

**예상 소요 시간**: 1일

### 작업 목록

- [ ] `components/album-card.tsx`를 클라이언트 컴포넌트로 전환(또는 hover 전용 하위 컴포넌트 분리) — 마우스 위치 추적이 필요하므로 `"use client"` 경계 설정
- [ ] 마우스 위치 기반 tilt 로직 구현 — `onMouseMove` 이벤트로 카드 내 포인터 좌표를 계산해 `rotateX` / `rotateY` 값을 산출
- [ ] CSS `perspective` + `transform: rotateX() rotateY()` 적용 — 카드 또는 이미지 컨테이너에 3D 변환 적용, `transition`으로 자연스러운 복귀 애니메이션 처리
- [ ] `onMouseLeave` 시 원래 각도(0deg)로 복귀
- [ ] 과도한 회전 방지를 위한 각도 클램핑 (예: 최대 ±8~10deg 수준으로 제한)
- [ ] 호버 시 미세한 그림자/하이라이트 강조 추가 (예: `box-shadow` 또는 `bg-gradient` 오버레이로 입체감 보강)
- [ ] 모바일/터치 디바이스 대응 — 터치 환경에서는 tilt 효과를 비활성화하거나 정적 호버로 대체 (`@media (hover: hover)` 또는 포인터 타입 체크)
- [ ] 접근성/성능 고려 — `prefers-reduced-motion` 사용자에게는 tilt 트랜지션을 생략

### 완료 기준

- 데스크탑에서 마우스를 카드 위에서 움직이면 마우스 위치에 따라 카드가 입체적으로 기울어진다
- 마우스가 카드를 벗어나면 원래 평면 상태로 자연스럽게 복귀한다
- 회전 각도가 과도하지 않아 카드 내용(이미지, 텍스트)이 시각적으로 깨지지 않는다
- 터치 디바이스(모바일)에서는 tilt로 인한 레이아웃 깨짐이나 의도치 않은 동작이 없다
- `prefers-reduced-motion: reduce` 설정 시 애니메이션이 최소화된다
- 다크모드와 라이트모드 양쪽에서 호버 효과(그림자/하이라이트)가 자연스럽게 보인다
- 기존 앨범 카드의 추천 배지·별점·텍스트 렌더링 로직에 회귀(regression)가 없다
